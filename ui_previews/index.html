UI Idea Lab - 네이버 모니터링 대시보드 후보

UX + 마케팅 + 개발 토론
UX 전문가

현재 표는 강력하지만 조회 시간이 많아질수록 사용자가 오늘 봐야 할 변화량을 놓치기 쉽습니다. 기준 시점, 판매 감소, 실패 재조회가 첫 화면에서 더 분명해야 합니다.

마케터

수강생에게는 “경쟁사가 얼마나 팔렸는지”가 가장 큰 소구점입니다. 단순 재고표보다 판매 추정 TOP, 재입고 감지, 성공률을 앞세우면 가치가 바로 전달됩니다.

개발자

실제 적용은 기존 renderKPIs, renderTable, productThumb 구조를 살리는 후보가 안전합니다. 과한 화면 전환보다 표 개선부터 가는 편이 리스크가 낮습니다.

A. 실전형 테이블 정돈

기존 대시보드와 가장 가까운 방향입니다. 수강생이 지금 쓰는 표 흐름은 유지하면서 KPI, 시간축, 옵션 상세를 더 읽기 쉽게 정리하는 안입니다.
